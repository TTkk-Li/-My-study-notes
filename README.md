## 网络协议与技术 arp icmp 、域名、dns、cdn

#### 一、ARP协议

根据目标IP地址，找到对应的MAC地址；



#### 二、ICMP协议

用于在IP主机、路由器之间传递控制信息；



#### 三、域名

由一串点分隔的名字组成的Internet上某一计算机或计算机组的名称；



#### 四、DNS和CDN

dns主要用于解析域名  ip  ———— 域名 之间相互转换；

CDN (Content Delivery Network) 内容分发网络，缓存主用户B内容。当用户请求该用户B数据时DNS会将请求重定向到距离用户物理位置最近的CDN边缘节点，缩短了物理距离、提升了响应速度；

## 传输层核心协议与DOS攻击

### 一、TCP工作原理

**三次握手**

 **第一次握手**
  客户端 → 服务器：发送 **SYN=1**，请求建立连接。

 **第二次握手**
  服务器 → 客户端：回复 **SYN=1, ACK=1**，同意建立连接，并确认收到客户端的请求。

 **第三次握手**
  客户端 → 服务器：发送 **ACK=1**，确认收到服务器的同意，**连接正式建立**。

```
核心作用：双方确认「自己能发、能收；对方能发、能收」。**四次挥手**
```

 **第一次挥手**
  主动关闭方（如客户端）→ 对方：发送 **FIN=1**，表示**我没有数据要发了，请求关闭**。

 **第二次挥手**
  被动关闭方 → 主动方：回复 **ACK=1**，表示**收到关闭请求**（但可能还有数据在传）。

 **第三次挥手**
  被动关闭方 → 主动方：发送 **FIN=1, ACK=1**，表示**我也发完了，可以关闭**。

 **第四次挥手**
  主动关闭方 → 对方：回复 **ACK=1**，确认收到，等待超时后**彻底断开连接**。



## 二、DDos攻击

DDoS是一种通过海量数据来强袭占用服务器运行资源的攻击手段。有一下三种类型

| 攻击类型               | 攻击层面 | 特点                |
| ------------------ | ---- | ----------------- |
| SYN 洪水攻击           | 传输层  | 占满服务器半连接队列        |
| UDP 洪水攻击           | 传输层  | 海量垃圾包占满宽带，服务器盲目处理 |
| ICMP 洪水攻击（Ping 洪水） | 网络层  | 海量 ICMP 包消耗服务器资源  |
